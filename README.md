# 📁 文件检测工具

一个简单易用的批量文件完整性检测工具，支持交互式操作和自动化检测，可打包为独立EXE程序。

## ✨ 功能特点

- ✅ **批量检测**：一次性检测大量文件的可读性和完整性
    
- ✅ **交互式操作**：双击运行，无需命令行参数，图形化交互界面
    
- ✅ **多线程处理**：快速完成大量文件检测，支持自定义线程数
    
- ✅ **详细报告**：自动生成文本和JSON格式的检测报告
    
- ✅ **问题识别**：清晰列出所有问题文件及错误原因
    
- ✅ **灵活配置**：支持扩展名过滤、递归检测等选项
    
- ✅ **独立运行**：可打包为EXE，无需安装Python环境
    

## 🚀 快速开始

### 方法一：直接使用EXE（推荐）

1. 下载已打包的 `文件检测工具.exe`
    
2. 双击运行即可弹出控制台界面
    
3. 按照提示选择检测模式
    

### 方法二：从源代码运行
```
base
# 克隆或下载源代码
git clone <仓库地址>

# 安装依赖
pip install -r requirements.txt

# 运行程序
python interactive_file_checker.py
```
## 📋 检测模式

程序提供四种检测模式：

1. **检测单个目录** - 检查指定目录下的所有文件
    
2. **检测多个目录** - 批量检查多个目录
    
3. **从文件列表读取** - 从TXT文件读取文件路径列表
    
4. **直接输入文件路径** - 手动输入要检测的文件
    

## ⚙️ 配置选项

- **线程数**：1-16个线程，根据CPU性能调整
    
- **扩展名过滤**：只检测特定类型的文件（如 .pdf .docx .jpg）
    
- **递归检测**：是否检查子目录中的文件
    
- **图标自定义**：可自定义程序图标（需要重新打包）
    

## 📊 输出结果

检测完成后会生成以下文件：

|文件类型|格式|用途|
|---|---|---|
|文本报告|`.txt`|易于阅读，包含问题文件列表|
|JSON报告|`.json`|便于程序处理，包含完整数据|

报告内容包括：

- 检查时间统计
    
- 总文件数和检测用时
    
- 正常/问题文件数量
    
- 问题文件详细列表（文件路径、错误原因、文件大小）
    

## 🔧 技术细节

### 文件检测原理

程序通过以下方式检测文件：

1. **存在性检查**：确认文件是否存在
    
2. **大小检查**：确认文件大小不为0
    
3. **可读性检查**：尝试读取文件头部数据
    
4. **完整性检查**：验证文件格式和结构
    

### 支持的文件类型

- 所有二进制文件（.exe, .dll, .zip, .rar等）
    
- 所有文本文件（.txt, .py, .java, .cpp等）
    
- 文档文件（.pdf, .docx, .xlsx, .pptx）
    
- 图片文件（.jpg, .png, .gif, .bmp）
    
- 多媒体文件（.mp3, .mp4, .avi, .mkv）
    

## 🐛 常见问题

### Q1: 程序运行时提示"权限不足"

**A**: 以管理员身份运行程序，或检查文件权限设置。

### Q2: 检测网络文件速度很慢

**A**: 网络文件检测受网络速度影响，建议减少并发线程数。

### Q3: 图标显示不正常

**A**: 确保图标是标准的.ico格式，且包含16x16和32x32尺寸。

### Q4: 打包时出现图标格式错误

**A**: 使用 `fix_build_exe.py` 脚本处理图标兼容性问题。

### Q5: 如何检测隐藏文件？

**A**: 程序默认会检测所有文件，包括隐藏文件。

### Q6: 检测报告文件太大

**A**: JSON报告包含完整数据，可定期清理或只保留文本报告。

## 📈 性能建议

- **少量文件**：使用4-8个线程
    
- **大量文件**：使用8-16个线程，分批次检测
    
- **网络文件**：使用2-4个线程，避免网络拥堵
    
- **SSD硬盘**：可使用更多线程加速检测
    

## 🔒 安全性

- 程序只读取文件，不会修改任何内容
    
- 不收集或上传用户数据
    
- 可完全离线使用
    
- 源代码开放，可自行审查
